API

IVT Playground API

Краткая документация по HTTP и WebSocket API текущего backend.

  • Сырой OpenAPI YAML: /api/openapi.yaml
  • Все защищённые HTTP-методы используют заголовок Authorization: Bearer <jwt>
  • Live-обновления приходят по /ws?token=<jwt>
Endpoints

Основные точки входа

GET /api/health

Проверка доступности backend.

POST /api/auth/login

Вход преподавателя или студента. Возвращает token, catalog, runtime.

GET /api/bootstrap

Восстановление текущей сессии по JWT.

POST /api/reset

Сброс стенда к seed-состоянию. Доступно преподавателю и администратору.

POST /api/actions/{action}

Универсальная точка входа для UI-команд.

GET /ws

WebSocket live-обновлений runtime и catalog.
